# Support

This repository is part of the KIM3310 portfolio business-activation surface.

## Public support

Use GitHub Issues for non-sensitive bugs, documentation problems, and feature requests. Do not post secrets, customer data, personal data, payment details, logs with tokens, or regulated data in public issues.

## Paid pilot / business inquiries

Use the paid-pilot intake issue template for non-sensitive scoping only. Customer names, contracts, production data, and pricing discussions should move to an owner-approved private channel before work begins.

## Privacy or data requests

Do not put personal data in a public issue. Use the privacy/data request template only to request a private contact path or to report that a privacy request exists. The owner must provide an approved private support/privacy channel before collecting personal data.

## Response expectations

- Free/demo/community requests: best effort.
- Paid pilot requests: first response target is 1 business day after a private support channel and paid scope are approved.
- Security-sensitive reports: do not disclose publicly; use the private channel specified by the owner.

Generated: 2026-06-26. This support policy is operational guidance, not a legal SLA.
